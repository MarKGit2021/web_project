27.03.22 примерный вариант

Сущности:

1. Пользователь:

[  ] Id

[  ] Телефон

[  ] Почта

[  ] Пароль

[  ] Имя

[  ] Фамилия 

[  ] Отчество

[  ] Тип пользователя (доверие пользователю) (0 - только читать/добавлять,  1 - исправлять, 2 - админ)

[  ] Количество лайков/баллов

Телефон или почта будут использоваться в качестве логина, отчество вводится по желанию. Тип пользователя определяется по количеству лайков/ баллов и отсутствию жалоб. (Можно ещё учитывать количество жалоб). Можно добавить столбец - является ли пользователь админом - и сделать для админа привилегии. Или сделать отдельную сущность для админа. 

2. Информация:

[  ] Id 

[  ] Html файл (ссылка/расположение/ название файла)

[  ] Id пользователя, который добавил информацию

[  ] Время создания

[  ] Id пользователя, который последний менял информацию

[  ] Время изменения

[  ] Информация заблокирована (да/нет) 

3. Комментарий

[  ] Id

[  ] Id пользователя, который написал

[  ] Id информации, куда написали

[  ] Дата добавления

[  ] Комментарий заблокирован (да/нет) 

4. Слова поиск:

[  ] Id 

[  ] Слово 

5. Запросы по слову (Информация по слову)

[  ] Id 

[  ] Id информации

[  ] Id слова

[  ] Популярность этого запроса (сколько раз этот запрос спрашивали)

Можно на главной странице выводить топ-10 самых популярных запросов 

6. История запросов 

[  ] Id

[  ] Id пользователя 

[  ] Id информации по слову

[  ] Дата запроса

[  ] Тип (изменил/добавил/прочитал/поискал) 

7. Жалоба

[  ] Id

[  ] Id информации по слову (на что жалуемся)

[  ] Id пользователя (кто жаловался)

[  ] Дата

[  ] Документ с жалобой (ссылка/расположение/название/прямо так, текстом)
